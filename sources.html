Method

Sources, verification, and weighting model

We intentionally blend official Microsoft updates with independent implementation evidence to reduce blind spots and increase operational usefulness.

Source mix target

  • ~60% independent/community technical analysis
  • ~40% official Microsoft announcements and documentation

Independent/official ratio can flex per week depending on signal quality.

Verification labels

  • Official — primary Microsoft source
  • Reputable community — known expert with reproducible detail
  • Early signal — plausible but requires validation

Change control

Tracked sources are maintained in data/sources.json. Adding/removing tracked feeds requires human approval before merge.
